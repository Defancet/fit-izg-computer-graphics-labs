Your assignment is to use GLSL language and create object/animation that the teacher will asign to you. You will use only GLSL language. Everything will be procedural. The goal of the Lab is to familiarize with the GLSL language in Open\+GL / Vulkan.\hypertarget{index_what}{}\doxysection{You should edit only these files\+:}\label{index_what}
shaders/student.\+glsl\hypertarget{index_theory}{}\doxysection{Theory}\label{index_theory}
GLSL is language for describing effects and programming shaders. Shader and shader program is code that can be executed on GPU. The purpose of a shader is to move with object, color them, create effects, compute physics, ... There are several types of shaders\+: 
\begin{DoxyItemize}
\item {\bfseries{Vertex Shader}} -\/ computes a vertex location. Usually multiplies vertices with matrix. 
\item {\bfseries{Fragment Shader}} -\/ Also known as pixel shader. Computes fragments. Usually computes color. 
\item Geometry Shader -\/ computes primitives 
\item Tessellation Shaders -\/ two shaders for primitive tessellation. 
\item Compute Shader -\/ for general computation. 
\item Other shaders -\/ non standardised shaders, extensions (ray-\/tracing, primitive shaders, ...) 
\end{DoxyItemize}

\hypertarget{index_how}{}\doxysection{How to do it\+:}\label{index_how}
The archive contains pre built Linux and Windows binaries.~\newline
 Archiv obsahuje zkopilované binárky pro Linux a pro Windows (shader\+\_\+lab\+\_\+linux.\+bin, shader\+\_\+lab\+\_\+windows.\+exe).~\newline
 If you can launch these binary, you don\textquotesingle{}t have to compile anything!~\newline
 If you cannot launch pre built binary, you have to compile the app by yourself, see \mbox{\hyperlink{index_compilation}{Compilation\+:}}.~\newline
 The application draws triangles (calls the commands gl\+Draw\+Arrays).~\newline
 Shaders inside \char`\"{}shaders/student.\+glsl\char`\"{} are executed. ~\newline
 File student.\+glsl contains section of vertex shader and framgment shader marked with preprocessor.~\newline
 You have to implement given effect/object/animation.~\newline
 You have in your disposal the counting number of vertex\+: \char`\"{}gl\+\_\+\+Vertex\+ID\char`\"{}.~\newline
 You can use this number to compute triangles and colors.~\newline
 Use attached shaders as an inspiration (directory shaders/).~\newline
 GLSL language specification\+: \href{https://www.khronos.org/registry/OpenGL/specs/gl/GLSLangSpec.4.60.pdf}{\texttt{ GLSL}}~\newline
 If you make a mistake, pop-\/up error message will be shown.~\newline
 Vertex shader is executed on every vertex, it\textquotesingle{}s purpose is to compute correct location.~\newline
 Fragment shader is executed on every fragment, it\textquotesingle{}s purpose is to compute color.~\newline
 The application is rendering quite a lot of triangles, you just need to use few of them and the rest can be degenerated to zero (setting gl\+\_\+\+Position=vec4(0)).~\newline
